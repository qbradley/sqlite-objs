### Phase 2: Critical Issues Found

**Issue C1 (HIGH PRIORITY):** Device Capabilities Flags
- **File:** `sqlite_objs_vfs.c:693`
- **Problem:** Claims `SQLITE_IOCAP_ATOMIC512 | SQLITE_IOCAP_SAFE_APPEND`
- **Risk:** SQLite assumes atomic multi-page writes; our sync is page-at-a-time. Data corruption.
- **Fix:** Replace with `SQLITE_IOCAP_SEQUENTIAL | SQLITE_IOCAP_POWERSAFE_OVERWRITE | SQLITE_IOCAP_SUBPAGE_READ`
- **Impact:** Architectural — must fix before any demo

**Issue C2 (HIGH PRIORITY):** URL Buffer Overflow
- **File:** `azure_client.c:173–187`
- **Problem:** `strcat` assembly of URLs with unbounded SAS tokens
- **Risk:** Stack buffer overflow with long tokens
- **Fix:** Replace with bounds-checked `snprintf` (4KB buffer)
- **Impact:** Security — must fix before production use
